使用说明

线上消息仍通过 Kafka 进入 PushService；管理台直接复用相同 Store、APNs Sender、双环境路由、重试和 token 失效处理。

PushKit 测试只验证 APNs/PushKit 通道，不会创建 CallService session。客户端被唤醒后调用 ResumeCall 会得到 call not found。
正文会进入 APNs，请勿填写敏感数据。
此操作可能直接拉起目标设备的 CallKit 来电界面，但服务端不存在对应通话 session。
